
1. FORM MANAGEMENT WORKFLOW

FormController
/api/v1/forms

Connected Services:
FormService - Core form CRUD, versioning, publishing
FormRenderService - Render forms for end users
SubmissionService - Handle submissions
UserRoleService - Role-based access control
AuditLogService - Log form operations
WorkflowService - Manage form workflows

Key Endpoints:
GET /api/v1/forms - List forms (role-based filtering)
POST /api/v1/forms - Create form
PUT /api/v1/forms/{id} - Update form
DELETE /api/v1/forms/{id} - Archive/delete form
GET /api/v1/forms/{id}/render - Public form rendering
POST /api/v1/forms/{id}/submit - Public form submission
POST /api/v1/forms/{id}/publish - Publish form
GET /api/v1/forms/{id}/versions - Get form versions

2. SUBMISSION MANAGEMENT WORKFLOW

SubmissionController
/api/v1/runtime (also /api/v1/forms)

Connected Services:
SubmissionService - Core submission operations (save, retrieve, update status)
FormService - Validate form exists and is published
UserRoleService - Check user permissions for submissions
AuditLogService - Log submission events

Key Endpoints:
GET /api/v1/runtime/forms/{id}/render - Get form for submission
POST /api/v1/runtime/forms/{id}/submit - Submit form data
GET /api/v1/runtime/forms/{idOrCode}/submissions - List submissions
GET /api/v1/runtime/forms/{id}/submissions/{submissionId} - Get submission details
PUT /api/v1/runtime/forms/{id}/submissions/{submissionId} - Update submission
DELETE /api/v1/runtime/forms/{id}/submissions/{submissionId} - Delete submission

3. CUSTOM VALIDATION WORKFLOW

CustomValidationRuleController

Key Endpoints:
GET /api/v1/forms/{formId}/versions/{versionId}/custom-validations - List validation rules
POST /api/v1/forms/{formId}/versions/{versionId}/custom-validations - Create rule
PUT /api/v1/forms/{formId}/versions/{versionId}/custom-validations/{ruleId} - Update rule
DELETE /api/v1/forms/{formId}/versions/{versionId}/custom-validations/{ruleId} - Delete rule

7. AUTHENTICATION WORKFLOW

AuthController
/api/v1/auth

Services Used: None (direct repository usage)

Connected Components:
UserRepository - RBAC user management
RoleRepository - Role assignments
PasswordEncoder - BCrypt password hashing

Key Endpoints:
POST /api/v1/auth/register - User registration (assigns Viewer role)
POST /api/v1/auth/login - Form-based login (Spring Security)
POST /api/v1/auth/logout - Session logout

8. RBAC - ROLE MANAGEMENT WORKFLOW

RoleController
/api/v1/roles

Connected Services:
RoleService - CRUD roles and permissions
AuditLogService - Log role changes

Key Endpoints:
GET /api/v1/roles - List all roles (system + custom)
POST /api/v1/roles - Create custom role
PUT /api/v1/roles/{id} - Update role permissions
DELETE /api/v1/roles/{id} - Delete custom role
POST /api/v1/roles/{id}/permissions - Add permissions to role

9. RBAC - USER MANAGEMENT WORKFLOW

UserController
/api/v1/users

Connected Services:
UserRoleService - Assign/remove roles from users
AuditLogService - Log user role changes

Key Endpoints:
GET /api/v1/users - List users (filtered by requesting user's role)
POST /api/v1/users - Create RBAC user profile
PUT /api/v1/users/{id} - Update user details
DELETE /api/v1/users/{id} - Delete user
POST /api/v1/users/{id}/roles - Assign role to user
DELETE /api/v1/users/{id}/roles/{roleId} - Remove role from user
GET /api/v1/users/{id}/permissions - Get user's effective permissions

13. WORKFLOW ENGINE (Form Approval) WORKFLOW

WorkflowController
/api/v1/workflows

Connected Services:
WorkflowService - Manage workflow approval steps
UserRoleService - Check reviewer permissions
AuditLogService - Log workflow transitions

Key Endpoints:
POST /api/v1/workflows/initiate - Start workflow for form
POST /api/v1/workflows/{instanceId}/step/{stepId}/approve - Approve step
POST /api/v1/workflows/{instanceId}/step/{stepId}/reject - Reject step
GET /api/v1/workflows/{formId}/status - Get workflow status

AdminWorkflowStatusController
/api/v1/admin/workflows

Connected Services:
WorkflowService - Query workflow instances
UserRoleService - Verify admin role

Key Endpoints:
GET /api/v1/admin/workflows/status - Monitor all form workflows (admin only)
